# 개인 지식 관리 및 할 일 자동화 시스템을 위한 도구 분석

사용자가 언급한 5가지 핵심 도구와 기술의 특징을 분석하여 시스템 구축의 기초로 삼습니다.

| 도구 | 핵심 역할 | 주요 기능 및 특징 | 통합 시 가치 |
| :--- | :--- | :--- | :--- |
| **OpenClaw** | **연동 및 실행 게이트웨이** | 이메일(Gmail), 메시징(Telegram, WhatsApp, Slack), 캘린더 등 다양한 외부 서비스와의 강력한 연동 API 제공. | 다양한 채널(이메일, 슬랙 등)로부터 데이터를 수집하고 실행 결과를 다시 전송하는 통로 역할. |
| **LLMWIKI** | **구조화된 지식 저장소** | Andrej Karpathy가 제안한 패턴으로, 비정형 데이터를 LLM이 읽기 쉬운 구조화된 Markdown 위키 형태로 자동 정리. | 수집된 파편화된 정보를 주제별로 정리하고 상호 참조(Cross-reference)가 가능한 지식 베이스 구축. |
| **Obsidian** | **사용자 인터페이스 및 로컬 저장소** | 로컬 Markdown 기반의 지식 관리 도구. 강력한 플러그인 생태계와 그래프 뷰 제공. | 사용자가 직접 지식을 열람, 수정하고 AI가 생성한 '할 일'과 '지식'을 시각적으로 확인하는 메인 허브. |
| **Hermes Agent** | **개인화된 메모리 레이어** | `USER.md`(사용자 성향)와 `MEMORY.md`(환경/맥락)라는 압축된 메모리 파일을 통해 장기 기억 및 개인화된 응답 구현. | 사용자의 선호도(예: "회장님께는 정중하게")와 현재 프로젝트 맥락을 기억하여 맞춤형 할 일 추천. |
| **Graphify** | **지식 그래프 및 관계 추출** | 텍스트 데이터에서 개체 간의 관계를 추출하여 그래프 구조로 시각화 및 검색 최적화. | 인물(회장, 부회장), 프로젝트, 할 일 간의 복잡한 관계를 연결하여 "누가 무엇을 해야 하는지" 명확히 파악. |

## 통합 시나리오
1. **수집**: OpenClaw가 이메일, 슬랙, 웹사이트 등에서 데이터를 긁어옵니다.
2. **정리**: LLMWIKI 패턴을 사용하여 데이터를 주제별 Markdown으로 변환합니다.
3. **연결**: Graphify가 데이터 간의 관계(인물-업무-지식)를 추출하여 연결합니다.
4. **개인화**: Hermes Agent가 사용자의 스타일과 현재 맥락을 반영하여 우선순위를 정합니다.
5. **표시**: 모든 결과물은 Obsidian에 저장되어 사용자가 언제든 확인하고 관리할 수 있습니다.
