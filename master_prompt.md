# 쟈비스 프로젝트: 개인 지식 관리 및 할 일 자동화 시스템 개발 마스터 프롬프트

## 1. 프로젝트 개요

당신은 **Manus AI**입니다. 사용자(저)를 위한 **LLM 기반 개인 지식 관리 및 할 일 자동화 시스템인 "쟈비스 프로젝트"**를 개발해야 합니다. 이 시스템은 다양한 데이터 소스(MD/PDF/TXT 파일, 이메일, 웹사이트, Telegram/Slack)를 통합하여 지식을 구조화하고, 사용자의 맥락과 선호도를 기반으로 "오늘/이번 주/이번 달 할 일" 및 "팀원(회장, 부회장)에게 보고하거나 지시할 사항"을 자동으로 도출하는 것을 목표로 합니다.

## 2. 핵심 요구사항 및 개발 원칙

*   **LLM 한계 극복**: LLM의 환각, 불안정한 출력, 긴 문맥 처리의 한계 등을 극복하기 위한 아키텍처 전략(RAG, 검증 레이어, 구조화된 출력 강제, 압축된 메모리 등)을 철저히 준수해야 합니다.
*   **모듈형 아키텍처**: 시스템은 **데이터 수집, 지식 처리, 지능형 메모리 및 추론, 실행 및 알림**의 4가지 핵심 모듈로 구성되어야 합니다. 각 모듈은 독립적으로 개발 및 배포가 가능하며, 표준화된 API(RESTful 또는 메시지 큐)를 통해 유기적으로 연동되어야 합니다.
*   **오픈소스 활용**: 제안된 핵심 오픈소스(OpenClaw, LLMWiki, Graphify, Mem0, CrewAI, Obsidian)를 적극적으로 활용하여 개발해야 합니다. GitHub에서 검증된(Star 수, 업데이트 빈도) 오픈소스를 우선적으로 고려합니다.
*   **유지보수 및 확장성**: 향후 기능 추가 및 기술 스택 변경에 유연하게 대응할 수 있도록 모듈 간의 결합도를 낮추고, 명확한 인터페이스를 정의해야 합니다.
*   **단계별 개발 및 검증**: 각 모듈은 독립적으로 개발하고, 개발 완료 시마다 철저한 테스트 및 검증을 거쳐야 합니다. (이 부분은 별도의 개발 가이드에서 상세히 지시할 예정입니다.)

## 3. 시스템 아키텍처 (Graphify 중심)

이전에 제안된 **Graphify 중심의 모듈형 시스템 아키텍처**를 기반으로 개발을 진행합니다. 상세 내용은 `/home/ubuntu/system_proposal_modular.md` 파일을 참조하십시오. 특히, `Graphify`는 모든 입력 데이터에서 핵심 개체와 관계를 정밀하게 추출하여 통합 지식 그래프를 구축하는 핵심 엔진 역할을 수행합니다.

## 4. 핵심 오픈소스 및 역할

| 기능 레이어 | 추천 오픈소스 | 역할 |
| :--- | :--- | :--- |
| **데이터 수집 (Ingestion)** | **[OpenClaw](https://github.com/OpenClaw/OpenClaw)** | 이메일, 텔레그램, 슬랙 등 멀티 채널 연동 게이트웨이 |
| **지식 구조화 (Wiki)** | **[LLMWiki (Karpathy Pattern)](https://github.com/lucasastorian/llmwiki)** | 비정형 데이터를 구조화된 Markdown 위키로 자동 정리 |
| **지식 그래프 (Relation)** | **[Graphify](https://github.com/safishamsi/graphify)** | 텍스트 내 복잡한 관계 추출 및 통합 지식 그래프 구축 핵심 엔진 |
| **메모리/개인화 (Memory)** | **[Mem0](https://github.com/mem0ai/mem0)** | 사용자의 선호도와 과거 맥락을 기억하는 지능형 메모리 관리 |
| **워크플로우 (Orchestration)** | **[CrewAI](https://github.com/crewAIInc/crewAI)** | 멀티 에이전트 기반의 할 일 분석 및 도출 워크플로우 |
| **사용자 인터페이스** | **[Obsidian](https://obsidian.md)** | 로컬 Markdown 기반 지식 허브 및 대시보드 |

## 5. 개발 진행 방식

당신은 다음의 개발 가이드(`development_guide.md` 파일로 제공 예정)에 따라 **모듈별로 순차적으로 개발을 진행**해야 합니다. 각 모듈 개발 완료 시에는 사용자에게 보고하고 다음 단계로 진행하기 위한 승인을 받아야 합니다. 모든 코드와 설정 파일은 `/home/ubuntu/jarvis_project/` 디렉토리 내에 모듈별로 구분하여 저장해야 합니다.

## 6. 초기 환경 설정

개발을 시작하기 전에, `/home/ubuntu/jarvis_project/` 디렉토리를 생성하고, 필요한 기본 환경(예: Docker, Python 가상 환경)을 설정해야 합니다. 각 오픈소스의 설치 및 초기 설정은 해당 프로젝트의 GitHub README를 참조하십시오.
