# 쟈비스 프로젝트: 단계별 개발 실행 가이드 (Playbook)

이 문서는 Manus AI가 "쟈비스 프로젝트"를 성공적으로 개발하기 위한 단계별 지침을 제공합니다. 각 단계는 모듈별 개발, 테스트, 그리고 사용자(저)의 검수 및 승인 절차를 포함합니다. 이 가이드를 철저히 준수하여 LLM 기반 시스템의 안정성과 확장성을 확보해야 합니다.

## 1. 개발 환경 설정

1.  **프로젝트 디렉토리 생성**: `/home/ubuntu/jarvis_project/` 디렉토리를 생성합니다.
    ```bash
    mkdir -p /home/ubuntu/jarvis_project
    cd /home/ubuntu/jarvis_project
    ```
2.  **기본 환경 설정**: 필요한 경우 Docker, Python 가상 환경 등을 설정합니다. 각 오픈소스의 설치 및 초기 설정은 해당 프로젝트의 GitHub README를 참조합니다.
3.  **버전 관리 초기화**: Git 저장소를 초기화하고, 각 모듈 개발 완료 시 커밋을 수행합니다.

## 2. 모듈별 개발 지침

각 모듈은 독립적인 서브 디렉토리 내에서 개발되어야 하며, 명확한 README.md 파일과 테스트 코드를 포함해야 합니다. 개발 완료 시에는 사용자에게 보고하고 다음 모듈 개발로 진행하기 위한 승인을 받아야 합니다.

### 2.1. 데이터 수집 모듈 (Data Ingestion Module) 개발

*   **목표**: 다양한 외부 소스(MD/PDF/TXT 파일, 이메일, 웹사이트, Telegram/Slack)로부터 원시 데이터를 수집하고 정규화하는 API를 구현합니다.
*   **세부 지침**:
    *   `/home/ubuntu/jarvis_project/ingestion_module/` 디렉토리를 생성합니다.
    *   **OpenClaw**를 설치하고, 이메일(Gmail API), Telegram, Slack 연동을 위한 기본 설정을 완료합니다.
    *   MD/PDF/TXT 파일 파싱 및 텍스트 추출 기능을 구현합니다. (예: `pypdf`, `python-docx`, `markdown` 라이브러리 활용)
    *   웹사이트 URL로부터 텍스트를 추출하는 크롤링 기능을 구현합니다. (예: `requests`, `BeautifulSoup` 활용)
    *   수집된 데이터를 정규화된 형식(예: JSON)으로 변환하는 API 엔드포인트를 개발합니다.
    *   LLM을 활용하여 초기 데이터 분류 및 필터링(예: 스팸 제거, 중요도 분류) 기능을 추가합니다.
*   **테스트**: 각 데이터 소스별로 샘플 데이터를 입력하여 정확하게 수집 및 정규화되는지 테스트합니다.
*   **보고**: 개발 완료 시, 사용자에게 `ingestion_module`의 기능과 테스트 결과를 보고합니다.

### 2.2. 지식 처리 모듈 (Knowledge Processing Module) 개발

*   **목표**: 수집된 원시 데이터를 가공하여 구조화된 지식으로 변환하고, Graphify를 활용하여 지식 그래프를 구축하는 API를 구현합니다.
*   **세부 지침**:
    *   `/home/ubuntu/jarvis_project/knowledge_processing_module/` 디렉토리를 생성합니다.
    *   **LLMWiki (Karpathy Pattern)**를 구현하여 정규화된 텍스트 데이터를 주제별 Markdown 파일로 요약 및 정리하는 기능을 개발합니다.
    *   **Graphify**를 설치하고, LLMWiki로 구조화된 Markdown 파일 및 원본 데이터에서 핵심 개체와 그들 간의 복잡한 관계를 정밀하게 추출하여 지식 그래프를 구축하는 API를 개발합니다.
    *   구축된 지식 그래프를 저장하고 관리하는 기능을 구현합니다. (예: Neo4j, ArangoDB 등 그래프 데이터베이스 또는 파일 시스템)
    *   `spaCy` 또는 `NLTK`와 같은 NLP 라이브러리를 활용하여 텍스트 전처리 및 개체명 인식을 수행합니다.
*   **테스트**: 다양한 유형의 샘플 문서를 입력하여 LLMWiki가 정확하게 요약하고, Graphify가 개체와 관계를 올바르게 추출하여 지식 그래프를 생성하는지 테스트합니다.
*   **보고**: 개발 완료 시, 사용자에게 `knowledge_processing_module`의 기능과 테스트 결과를 보고합니다.

### 2.3. 지능형 메모리 및 추론 모듈 (Intelligent Memory & Reasoning Module) 개발

*   **목표**: 사용자의 맥락과 선호도를 기억하고, 지식 그래프를 기반으로 추론하여 실행 가능한 통찰을 도출하는 API를 구현합니다.
*   **세부 지침**:
    *   `/home/ubuntu/jarvis_project/memory_reasoning_module/` 디렉토리를 생성합니다.
    *   **Mem0**를 설치하고, 사용자의 프로필, 선호도, 과거 상호작용 기록 등을 관리하는 기능을 구현합니다.
    *   **RAG (Retrieval-Augmented Generation) 엔진**을 구현하여 Graphify 지식 그래프에서 필요한 정보를 검색하고, 이를 LLM의 프롬프트에 통합하는 기능을 개발합니다. (예: `LangChain`, `LlamaIndex` 활용)
    *   LLM을 활용하여 사용자 선호도 학습, 지식 그래프 질의 응답, 복잡한 추론 및 의사결정 지원 기능을 개발합니다.
    *   LLM의 환각을 줄이기 위한 검증(Verification) 레이어를 구현합니다.
*   **테스트**: 다양한 사용자 시나리오(예: 특정 주제에 대한 질문, 특정 업무에 대한 조언 요청)를 통해 Mem0가 맥락을 올바르게 기억하고, RAG 엔진이 정확한 정보를 검색하며, LLM이 합리적인 추론 결과를 도출하는지 테스트합니다.
*   **보고**: 개발 완료 시, 사용자에게 `memory_reasoning_module`의 기능과 테스트 결과를 보고합니다.

### 2.4. 실행 및 알림 모듈 (Action & Notification Module) 개발

*   **목표**: 추론 모듈에서 도출된 할 일 및 지시사항을 사용자에게 전달하고, 필요한 경우 외부 시스템에 액션을 수행하는 API를 구현합니다.
*   **세부 지침**:
    *   `/home/ubuntu/jarvis_project/action_notification_module/` 디렉토리를 생성합니다.
    *   **CrewAI**를 설치하고, 멀티 에이전트 워크플로우를 구성하여 "나의 할 일", "회장/부회장 지시사항", "팀 업무 배분" 등의 최종 리포트를 생성하는 기능을 개발합니다.
    *   **Obsidian Sync** 기능을 구현하여 생성된 할 일 목록 및 지식 그래프 시각화 대시보드를 Obsidian 로컬 저장소에 동기화합니다.
    *   **OpenClaw**를 재활용하여 Telegram/Slack 등 메신저를 통해 사용자에게 알림을 전송하는 게이트웨이를 구현합니다.
    *   LLM을 활용하여 최종 할 일 목록의 자연어 생성, 알림 메시지 작성, 팀원별 맞춤형 지시사항 생성 기능을 개발합니다.
*   **테스트**: 추론 모듈에서 생성된 할 일 및 지시사항이 정확하게 포맷팅되어 Obsidian에 동기화되고, 적절한 채널로 알림이 전송되는지 테스트합니다.
*   **보고**: 개발 완료 시, 사용자에게 `action_notification_module`의 기능과 테스트 결과를 보고합니다.
